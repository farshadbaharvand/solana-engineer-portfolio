# تنظیم دوره: پیکربندی محیط توسعه سولانا (Solana)

قبل از ورود به مبحث توسعه **قرارداد هوشمند (Smart Contract)**، ایجاد یک محیط توسعه پایدار بسیار حیاتی است. این درس به نصب ابزارهای ضروری، پیکربندی **رابط خط فرمان سولانا (Solana CLI)**، مدیریت کیف پول و روند کار با **اعتبارسنج بلاک‌چین محلی (Local Blockchain Validator)** می‌پردازد.

---

## پیش‌نیازها و نصب ابزارها

برای توسعه در شبکه سولانا، نیاز دارید سه مؤلفه اصلی را روی سیستم خود نصب کنید: **Rust**، **Solana CLI** و **Anchor CLI**.

برای دریافت دقیق‌ترین دستورالعمل‌های نصب متناسب با سیستم‌عامل خود (macOS، Linux یا Windows)، لطفاً به مستندات رسمی نصب سولانا (Solana Installation Documentation) مراجعه کنید.

### نسخه‌های نرم‌افزاری توصیه‌شده
برای اطمینان از سازگاری محیط شما با محتوای دوره و جلوگیری از بروز مشکلات احتمالی، توصیه می‌شود از نسخه‌های زیر استفاده کنید:

- **Rust**: 1.87.0
- **Solana CLI**: 3.0.10 یا 3.0.13
- **Anchor CLI**: 0.31.1

**نکته**: ممکن است در برخی آموزش‌های متفرقه، نسخه‌های قدیمی‌تری (مانند 0.29.0) را مشاهده کنید، اما این دوره برای نسخه 0.31.1 بهینه‌سازی شده است.

---

## تأیید نصب ابزارها

پس از نصب، ترمینال خود را باز کرده و دستورات زیر را برای تأیید نصب موفقیت‌آمیز و بررسی نسخه‌ها اجرا کنید:

```bash
rustc --version       # تأیید Rust
solana --version      # تأیید Solana CLI
anchor --version      # تأیید Anchor CLI
```

## پیکربندی Solana CLI

Solana CLI به عنوان مرکز کنترل محیط توسعه شما عمل می‌کند. این ابزار تعیین می‌کند که شما با کدام Cluster شبکه تعامل دارید؛ چه در حال استقرار در شبکه تست عمومی باشید و چه به صورت محلی کار کنید.


برای مشاهده تنظیمات پیکربندی فعلی خود، شامل **RPC URL**، **WebSocket URL** و مسیر فایل **جفت‌کلید (Keypair)**، دستور زیر را اجرا کنید:

```bash
solana config get
```

### تعیین هدف کلاستر (Cluster Target)
شما باید به صورت صریح به **CLI** اعلام کنید که تراکنش‌ها را به کجا ارسال کند:

- **Localhost**: برای توسعه و تست سریع محلی از این گزینه استفاده کنید.
- **Devnet**: برای استقرار برنامه‌های خود در شبکه تست عمومی از این گزینه استفاده کنید.

برای جابجایی بین این محیط‌ها از دستورات زیر استفاده کنید:

```bash
# تغییر به Localhost (برای تست محلی)
solana config set --url localhost
# نسخه کوتاه دستور:
solana config set -ul

# تغییر به Devnet (برای تست عمومی)
solana config set --url devnet
# نسخه کوتاه دستور:
solana config set -ud
```

## راه‌اندازی کیف پول فایل‌سیستمی (File-System Wallet)

برای امضای تراکنش‌ها و پرداخت هزینه‌های استقرار (Gas)، به یک کیف پول سولانا نیاز دارید. **CLI** کیف پول‌های "فایل‌سیستمی" را مدیریت می‌کند که کلید خصوصی شما را در یک فایل **JSON** روی کامپیوتر ذخیره می‌کند.

### تولید کیف پول جدید
برای ایجاد یک کیف پول پیش‌فرض جدید، اجرا کنید:

```bash
solana-keygen new
```

**نکته**: اگر قبلاً در مسیر پیش‌فرض (`~/.config/solana/id.json`) کیف پولی داشته باشید، **CLI** خطایی برمی‌گرداند تا از بازنویسی تصادفی کلیدهای شما جلوگیری کند. باید کیف پول قدیمی را جابجا کنید یا در صورت تمایل به بازنویسی، از فلگ اجبار (force flag) استفاده کنید.

### مشاهده جزئیات کیف پول
پس از تولید جفت‌کلید، می‌توانید آدرس عمومی و موجودی **SOL** خود را مشاهده کنید:

```bash
# مشاهده کلید عمومی (آدرس کیف پول)
solana address

# بررسی موجودی فعلی SOL
solana balance
```

## دریافت SOL تست‌نت از طریق ایردراپ (Airdrop)

برخلاف توسعه اتریوم که معمولاً نیاز به مراجعه به وب‌سایت‌های ثالث (Faucet) دارید، سولانا به شما اجازه می‌دهد توکن‌های تست را مستقیماً از طریق **CLI** درخواست کنید. این توکن‌ها برای پرداخت کارمزد تراکنش‌ها در **Devnet** ضروری هستند.

### روند کار با Airdrop
ابتدا اطمینان حاصل کنید که روی **Devnet** هستید:

```bash
solana config set -ud
```

سپس درخواست ایردراپ کنید:

```bash
# درخواست 2 واحد SOL
solana airdrop 2
```

**عیب‌یابی**: اگر خطای "Rate Limit" دریافت کردید، به این معنی است که شبکه در حال حاضر ترافیک بالایی دارد. لحظاتی صبر کنید و دوباره دستور را امتحان کنید تا موفقیت‌آمیز باشد.

## اجرای اعتبارسنج (Validator) محلی سولانا

برای بخش بزرگی از چرخه توسعه، به ویژه هنگام اجرای تست‌ها یا اسکریپت‌های استقرار به صورت محلی، باید از **Test Validator** استفاده کنید. این ابزار یک شبیه‌سازی با کارایی بالا از بلاک‌چین سولانا را مستقیماً روی ماشین شما اجرا می‌کند.

### نحوه شروع Validator
تنظیمات پیکربندی را روی **Localhost** قرار دهید:

```bash
solana config set -ul
```

سپس فرآیند را شروع کنید:

```bash
solana-test-validator
```

**نکته مهم در روند کاری**: وقتی دستور `solana-test-validator` را اجرا می‌کنید، این فرآیند پنجره ترمینال فعلی شما را اشغال می‌کند. این پنجره را نبندید. تا زمانی که آن پنجره باز باشد، بلاک‌چین محلی شما فعال است. برای اجرای دستورات دیگر (مانند استقرار کد)، باید یک تب یا پنجره ترمینال جدید باز کنید.

## درک ساختار دایرکتوری دوره

مخزن (Repository) دوره به گونه‌ای ساختار یافته است که از توسعه به صورت **Native Rust** و همچنین توسعه با **Anchor Framework** پشتیبانی کند. تمام تمرین‌ها در پوشه `apps` قرار دارند.

در اینجا سلسله‌مراتبی که با آن کار خواهید کرد آمده است:

```text
repo/
└── apps/
    ├── [app_name]/         # مثال: 'hello'
    │   ├── anchor/         # تمرینات با استفاده از Anchor Framework
    │   │   ├── exercise/   # <--- کد خود را اینجا بنویسید
    │   │   ├── solution/   # کد مرجع (اگر گیر کردید)
    │   │   └── README.md   # راهنمای اختصاصی برای این ماژول
    │   │
    │   └── native/         # تمرینات با استفاده از Native Rust
    │       ├── exercise/   # <--- کد خود را اینجا بنویسید
    │       ├── solution/   # کد مرجع
    │       └── README.md   # راهنمای اختصاصی برای این ماژول
```

---

## نکات کلیدی

- **انتخاب مسیر**: بسته به درس مربوطه، به پوشه `native` یا `anchor` بروید.
- **محیط کار**: همیشه کد خود را داخل دایرکتوری `exercise` بنویسید.
- **مراجع**: فقط در صورتی که نیاز دارید کار خود را با پیاده‌سازی صحیح مقایسه کنید، از پوشه `solution` استفاده کنید.
- **دستورالعمل‌ها**: همیشه فایل `README.md` موجود در پوشه اپلیکیشنِ خاص را برای راهنمایی مرحله‌به‌مرحله مطالعه کنید.
